# ChronoGym 🏋️⏱️

A full systems analysis and design project conceptualizing a 
real-time gym crowd tracking application, completed as part of 
an Information Systems course.

## Project Overview
ChronoGym addresses a common pain point for gym-goers, not knowing 
how busy a gym is before arriving. This project walks through the 
complete systems development lifecycle (SDLC) of a proposed mobile 
app that aggregates real-time and historical crowd data from gyms 
to help users plan their workouts smarter.

## Skills Demonstrated

- Requirements Gathering & Documentation
- Database Design (ERD)
- Data Flow Modeling (DFD & Context Diagrams)
- UI/UX Wireframing & Storyboarding
- Business Needs Assessment

## Key Features Designed
ChronoGym was designed to handle real-time crowd tracking with a 
3-tier capacity system, personalized gym recommendations based on 
user preferences, and a gym search engine with filters for 
equipment, courts, and crowd levels. It also includes user 
authentication, user-generated crowd reports as a data fallback, 
and a ratings and reviews system.

## Relevance to Data & Business
This project involved designing a system that handles multiple 
data sources such as gym-provided data, real-time user reports, 
historical trends, and presenting that data in a clean, actionable 
way for end users. The ERD reflects a normalized relational database 
structure spanning 15+ interconnected tables.
